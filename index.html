HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user. It is an example of a semantic HTML element, meaning the name of the element describes its conent and role on the wbepage.
  • The header element is represented by h1, h2, h3, etc. Headings are titles and subtitles that help organize a page.
  • Img is an example of an element contained in the header. Elements like img have attributes, attributes being additional information about an element. For example, some attributes of img define the source of the image file, or describing the apprearance or function of an image.
  • Any element inside of another element is called a child element. The element that holds all of the "children" is the parent element.
  • The section element is a semantic element that defines a section in a document that contains thematic content.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicate how much space we want around the content inside an element.
  • The three ways to style a webpage using CSS are: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Inline CSS is used when a developer wants to style an element directly in the HTML file. This is useful for quick and permanent changes.
  • Internal style sheet contains CSS rules for the webpage in the head section of the HTML file. The rules only apply to that page.
  • The most common way to style a webpage is by using an external CSS style sheet. It keeps all CSS rules in a separate file, making design changes easier.
  • A selector deines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two components: the CSS property we want to apply and the value of the property.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • The class attribute can be assigned to the element in the HTML, letting us choose which elements will be assigned the CSS rule. Example: the class selector, card

Git

  • Git is a version control system that lets you manage and keep track of versions of projects.
  • Every project on GitHub is called a repository, which is a folder for your proect that contains all files and their revision history.
  • git pull: fetch and merge any commits from the remote branch
  • git push: transmit local branch commits to the remote repo branch
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A will add changed files in our working directory to Git
  • git commit -m "description" will commit changes to the repo
  • git pull origin main makes sure our current branch is up to date before opening a PR
  • git push origin feature/nameofbranch to push all of the changes to Git
